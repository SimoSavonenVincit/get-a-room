# Development Rules for Meeting Room Booking API

## Process Rules
1. **Documentation**: Always update PROMPTS.md with user requests and response summaries
2. **Consultation First**: Ask clarifying questions before starting any task
3. **Best Practices**: Follow industry standards and clean architecture principles
4. **Test-Driven**: Write comprehensive tests alongside development
5. **Step-by-Step**: Proceed incrementally, consulting with user at each major step

## Testing Requirements
- Maintain test battery that runs with a simple command
- Ensure tests cover core functionality
- Make tests easy to understand and maintain

## Architecture Principles
- Use appropriate design patterns
- Maintain separation of concerns
- Write clean, maintainable code
- Document key decisions
